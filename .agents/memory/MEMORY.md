- [Settings persistence](settings-persistence.md) — platform_defaults + workspace_profile saved to public.settings (key/value JSONB, unique user_id+key); load via loadSettings(), save via saveSettings().
- [Content page patterns](content-page-patterns.md) — getSupabase() try/catch wrapper required in all "use client" components; use PLATFORM_BY_ID[id] not PLATFORM_REGISTRY[id as keyof]; platform name is cfg.ui.displayName not cfg.name.
- [Generate route enrichment](generate-route.md) — AI title/description rewritten once per URL (shared across platforms); scheduled_posts.metadata includes og_image, title, description, source_url, char_limit, hashtags, content_pending, generated_at.
- [Credential column mismatch](credential-column-mismatch.md) — /api/connections saves to credentials_encrypted (AES-256-GCM base64 JSON blob); publisher reads this column first, falls back to access_token_enc. Never assume access_token_enc is populated.
- [Content page persistence](content-persistence.md) — UrlCard loads existing scheduled_posts on mount (status pending/failed, content_pending=false) to restore generatedPosts state; char_limit stored in scheduled_posts.metadata so client can reconstruct GeneratedPost without importing server-only PLATFORM_LIMITS.
- [AI title generation](ai-title-generation.md) — rewriteTitle must use plain-text prompt (not JSON array via buildTitlePrompt); strip code fences + quotes from Gemini response; take first line only. Both generate/route.ts and campaign/auto-generate/route.ts have their own copy — keep in sync.
- [Social post URL inclusion](social-post-url.md) — Source URL appended to content in generate route for non-publishing platforms; PUBLISHING_PLATFORMS = {devto, hashnode, medium, substack} handle URL via their adapter body builders. maxHashtags in PlatformDefaultSettings caps hashtag count (0 = use platform default from PLATFORM_LIMITS).
